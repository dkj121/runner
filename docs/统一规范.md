# 统一规范

## Git 规范

|   topic   |     解释     |
| :-------: | :----------: |
|   main    |   主分支    |
| feature/* | 功能实现分支 |
| release/* |   发布分支   |
| hotfix/*  | bug 修复分支 |

```bash

 git-flow init -p=github      // 使用 GithubFlowPreset 来做 GitFlow 初始化
 git-flow topic start name    // 创建名为 name 的 type 分支
 git-flow topic finish name   // 结束名为 name 的 topic 分支
 git-flow topic public name   // 将名为 name 的 topic 分支发布到云端
 git-flow topic delete name   // 删除名为 name 的 topic 分支
 git-flow topic checkout name // 选中名为 name 的 topic 分支

```

## 网络路径规范

 路径名统一用名词复数后接动词，单词全小写，多个单词之间通过'_'链接。
 例如：https://example.com/users/update_run

## API 规范

### 设计流程依照

 `流程驱动 -> 数据建模 -> api 编写`

### TS 代码规范

1. 尽可能优先使用联合类型的参数而不是重载
2. 尽可能使用 const 'name' = {} as const 而不是 enum
