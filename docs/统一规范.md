# 统一规范

## Git 规范

所有的 PR（Pull Request）都要处作者外另外所有人认可~~（毕竟就仨）~~

|   topic    |     解释     |
| :--------: | :----------: |
|    main    |    主分支    |
| feature/\* | 功能实现分支 |
| release/\* |   发布分支   |
| hotfix/\*  | bug 修复分支 |

```bash

 git-flow init -p=github      // 使用 GithubFlowPreset 来做 GitFlow 初始化
 git-flow topic start name    // 创建名为 name 的 type 分支
 git-flow topic finish name   // 结束名为 name 的 topic 分支
 git-flow topic publish name   // 将名为 name 的 topic 分支发布到云端
 git-flow topic delete name   // 删除名为 name 的 topic 分支
 git-flow topic checkout name // 选中名为 name 的 topic 分支

```

## DB 规范

1. 完美适配 better-auth 最低要求
2. 关键信息保留 create_at/update_at 等必要时间戳
3. 常用字段添加索引

## API 规范

1. 与 DB 适配、不臃肿
2. 能从客户端聚合的信息不要丢给服务端
3. request / response 适配 DB 中的数据结构
